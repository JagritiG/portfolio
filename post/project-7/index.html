AWS Project: Performance Analysis and Speedup of AWS Data Pipeline and Analytics

June 20, 2021

According to Statista, worldwide data volume, from 2010 to 2025, are increasing rapidly – reaching 64.2 zettabytes in 2020 and projected to grow more than 180 zettabytes by 2025. With the rapid growth of the data volume, storage capacity is projected to increase, growing at a compound annual growth rate of 19.2% over the forecast period from 2020 to 2025.

With the exponential growth of data, organizations are facing many data analytics challenges such as:

  • Inefficient data management due to large size, complexity, variety, and variability
  • Data velocity – speed of data generation and process to meet demands
  • Open-source data analytics tools to meet enterprise security standard
  • Processing power – latency and cost of data processing

In this project, we will focus on performance analysis and speedup of AWS Data Pipeline and Analytics.

Why Choose Amazon Web Service (AWS) for Data Analytics?

Serverless capabilities of Amazon S3, AWS Glue, Amazon Athena, Amazon QuickSight, and AWS Lambda gives flexibility to build data pipeline and analytics architecture to speedup data analytics process. We can store, transform, enrich, analyze, and visualize a large volume of data using these AWS services in a highly durable, secure, and scalable manner. In the serverless data analytics environment, there is no infrastructure needed to manage and monitor the performance.

According to Gartner 2020 report, Amazon Web Service is leading with the highest score in both axes of measurement – Ability to Execute and Completeness of Vision. According to the report on cloud market share in 2021 by Statista, Amazon leads Cloud market share (32%) followed by Azure (20%), Google Cloud (9%), and many more behind them. Components of the above serverless data pipeline and analytics architecture are as follows:

Amazon S3

Amazon S3 or Amazon Simple Storage Service is an object storage service that offers industry-leading scalability, high availability, security, and low latency with 99.999999999% durability and between 99.95% to 99.99% availability. Amazon S3 can be used to store and retrieve any amount of data.

Amazon Glue

AWS Glue is a fully managed ETL (extract, transform, and load) service that makes it simple and cost-effective to categorize, clean, enrich, and move data reliably between various data stores and data streams. AWS Glue discovers your data and stores the associated metadata (e.g., table definition and schema) in AWS Glue Data Catalog. Once cataloged, data is immediately searchable, queryable, and available for ETL.

AWS Glue Data Catalog

The AWS Glue Data Catalog is a central metadata store. It is a managed service that lets you store, annotate, and share metadata in the AWS Cloud in the same way you would in an Apache Hive metastore.

AWS Glue Crawler

An AWS Glue Crawler reads data from data stores and writes to the targets specified in the source and target Data Catalog tables. A crawler can crawl multiple data stores in a single run. Upon completion, the crawler creates or updates one or more tables in Data Catalog.

AWS Glue ETL Job

Using the metadata in the Data Catalog, AWS Glue automatically generates a Scala or PySpark scripts, which runs on Apache Spark, to perform extract, transform, and load (ETL) work in AWS Glue.

Amazon Athena

Amazon Athena is an interactive query service that makes it easy to analyze data in Amazon S3 using standard SQL. Athena supports and works with a variety of standard data formats, including CSV, JSON, Apache ORC, Apache Avro, and Apache Parquet. Using Athena you can point to data in Amazon S3, define the schema, and start querying using standard SQL. Athena is out-of-the-box integrated with AWS Glue Data Catalog.

Amazon QuickSight

Amazon QuickSight is a fully managed business intelligence (BI) service. QuickSight lets you create and publish interactive dashboards that includes Machine Learning-powered insights. QuickSight dashboards can be accessed from any device, and embedded into your applications, portals, and websites.

Environment and Security Setup

  1. Create and activate an AWS account (instructions)
  2. Create an Amazon Identity and Account Management (IAM) Role to give
    1. S3 bucket access permission
    2. Glue service permission for crawling and transforming data
Environment and security setup steps:
    1. Go to IAM dashboard and click “Roles”
    2. Click on “Create Role”
    3. On the create role dashboard, select AWS service and choose Glue from the list of services
    4. Hit Next: Permission to continue
    5. From the Filter Policies list enter “glue” and select the AWSGlueServiceRole policy
    6. Enter “s3” into the Filter Policies list and select the AmazonS3FullAccess policy (in real project, policy is used to provide read access to the bucket)
    7. Enter “athena” into the Filter Policies list and select the AmazonAthenaFullAccess policy
    8. Enter “quicksight” into the Filter Policies list and select the AmazonQuicksightAthenaAccess policy
    9. Click Next: Tags
    10. Add any tags or leave it blank and click Next: Review
    11. Give your role a name (e.g., based on this project: sales_insights_s3_glu_athena) and click “Create Role”

Data

I have used open datasets from Kaggle in this project. Though our example datasets are small compared to most big data processing scenarios but they are enough to achieve our project goal.

  1. City Temperature (140MB): Daily temperature of major cities of the world. Source: kaggle
  2. Electronics Store (360MB): Purchase data from April 2020 to November 2020 from a large home appliances and electronics online store. Source: kaggle
  3. Customer complaints (891MB): Real world complaints data received about financial products and services. Source: kaggle

Methodology

1. Data storage in Amazon S3 bucket

  1. Create and configure Amazon S3 buckets
  2. Import CSV files into buckets

2. Data Extract - Transform - Load (ETL)

  1. Create Amazon Glue Crawler and crawl raw CSV data from Amazon S3 buckets
  1. Create Amazon Glue Data Catalog
  1. Transform raw CSV data to Parquet, crawl, and add parquet to Glue Data Catalog

Data size can be reduced by 80.46% by transforming raw CSV data into Parquet (around 5 to 1 compression), which results in further reduction in query cost.

3. Data Analysis

  1. Connect Amazon Athena with Glue Data Catalog
  2. Use standard SQL to query data with Athena

4. Visualization

  1. QuickSight Account Setup
Steps to QuickSight account setup:
1. Go to Amazon Management Console > type QuickSight on search box
2. Inside Services click on QuickSight
3. In the new window > click on Sign up for QuickSight
4. Select Enterprise version (Standard and Enterprise both are free for 60 days) and click on Continue
5. In the Create your QuickSight account screen, for Edition select Use IAM federated identities only (based on your project requirement)
6. In QuickSight region select region that you are using for other AWS service (e.g. US East (Ohio))
7. Enter QuickSight account name (e.g. Yourname-Athena)
8. Enter Notification email address (e.g. xxx@gmail.com)
9. Check Enable autodiscovery of data and users in your Amazon Redshift, Amazon RDS, and Amazon IAM services
10. Check Amazon Athena
11. Check Amazon S3 and select the buckets that you want QuickSight to be able to access and check Write permission for Athena Workgroup
12. Click Finish
13. In the new screen, click on Go to Amazon QuickSight
  1. Connect QuickSight with Athena and add data sources
  2. Create data model and interactive dashboard

Run SQL queries with Amazon Athena

Open the Athena dashboard and select the database to perform query. We run the following query on complaints data:

SELECT Count(*) FROM "complaints"."cust_complaints"
where "complaints"."cust_complaints".subproduct='Mortgage';

Query Performance

We have performed following two simple queries using MySQL and AWS Athena (both raw and parquet data) and compared query performance as shown in the below tables and plots.

  SELECT Distinct column FROM table;
  SELECT Count(*) FROM table where column='value';

Average data scanned is decreased by 99.5% with parquet data. Average runtime is reduced by 3.33x (from using MySQL) and 2.22x (from raw data using Athena) with parquet data.
