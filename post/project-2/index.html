Python Development Project: A Python DB API for Google Spreadsheets (0.2.0) - client project

January 14, 2021

As Google Sheet is web-based, one can access the data stored on Sheets from anywhere via the internet. Since it is a cloud-based app, one can use it as a database for small applications or websites, and can manage it in real-time. But, a very limited query languages are available to query Google sheets. A Python DB API for Google Spreadsheets version (<= 0.1.12) allows to query Google sheets, but does not support INSERT/UPDATE/DELETE operations. To remove these limitations, gsheets-db-api-plus (0.2.0) has been created by implementing Data Manipulation Language (DML) interface (insert, update, delete operations), which allows to query, insert, update, and delete Google Spreadsheets using SQL and SQLAlchemy grammar. When the insert, update, delete, and query statements are expressed, this module downloads Google sheet data into an in-memory SQLite table, and executes the statement/query in SQLite. After the execution of the given statement or query, the updated data are uploaded into the Google sheet.

Examples using SQLAlchemy grammar

INSERT:
engine.execute(table.insert(), column1=value1, column2=value2, column3=value3, ...)
UPDATE:
engine.execute(table.update().where(table.c.column == value).values(column1=value1, column2=value2, column3=value3,...))
DELETE:
engine.execute(table.delete().where(table.c.column <= value))
